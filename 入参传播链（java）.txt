### 分析策略
采用经典污点传播模型，并具备**跨文件与跨语言追踪能力**，重点关注：
- **Source**：参数如何进入方法（HTTP 请求、RPC 输入、方法参数、序列化数据等）
- **Propagation**：参数在方法内如何传递、拼接、转换（字符串操作、对象引用、IO 流、反射等）
- **Cross-Boundary**：**关键增强项**。当检测到 Java 调用外部脚本（Shell/Python 等）、读写配置文件或生成可执行文件时，**必须主动检索项目内相关文件内容**，继续分析数据流在外部文件中的走向。
- **Sanitizer**：是否存在校验/过滤逻辑（路径规范化、白名单、正则校验等）
- **Sink**：最终是否流向危险操作（命令执行、文件操作、数据库、反序列化、**外部脚本内的危险函数**等）

### 分析要求
- **跨文件追踪**：若 Java 代码引用了外部脚本（如 `Runtime.exec("python script.py")`）或操作特定文件，必须加载该文件内容分析其内部是否存在二次注入或逻辑绕过。
- **命令执行**：明确识别具体命令及参数构造方式。若命令指向脚本，**Sink 点可能位于脚本内部**（如脚本内的 `eval`、`system` 等）。
- **文件操作**：
  - 写文件：评估写入内容是否可控，是否会被后续逻辑执行（如写入 `.sh`、`.jsp`、配置文件）。
  - 读文件：评估文件内容是否被直接用于敏感操作（如反序列化、动态加载）。
- **校验逻辑**：翻译为自然语言描述，评估绕过可能性（路径穿越、正则绕过、竞争条件等）。
- **其他危险点**：SQL 注入、XXE、SSTI、反射滥用、不安全随机数、硬编码凭证等。
